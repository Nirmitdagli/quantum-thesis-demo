Tom & Jerry: Find the Key
Grover's Quantum Search Algorithm

Imagine Tom the cat is searching for Jerry the mouse, who's hiding in one of 16 rooms in a big house. Tom has no clue which room. How does each "computer" find Jerry?

CPU (Tom alone) — Opens doors one by one. Room 1... nope. Room 2... nope. Might check up to all 16 rooms. That's brute force — slow but guaranteed.

GPU (Tom + 3 friends) — Four cats split up and check 4 rooms at the same time! Done in 4 rounds instead of 16. That's parallel processing.

QPU (Quantum Tom) — Uses quantum magic to "listen" for Jerry through all doors at once. After just 3 sniffs (Grover iterations), Quantum Tom is 96% sure which room Jerry is in. That's quantum superposition + amplitude amplification.

All Three Together — CPU organizes the search plan, GPU builds a map of the house, QPU does the quantum sniffing. The dream team.

TL;DR: CPU checks rooms 1-by-1 (slow). GPU checks in batches (faster). QPU senses the answer in sqrt(N) steps (magical). Mix and match below to see the difference yourself!
Our demo: 16 rooms (4 qubits) Real world: AES-128 = 340,000,000,000,000,000,000,000,000,000,000,000,000 "rooms"

Security Guards: Catch the Hacker
Quantum Support Vector Machine (QSVM)

A huge concert has people entering through the gate. Half are real fans, half are hackers in disguise trying to sneak in. The security team needs to tell them apart.

CPU (One guard with a checklist) — Checks each person's ID one by one. Compares their face to a "suspicious patterns" database. Slow but catches everyone — 100% accuracy.

GPU (Team of guards) — Four guards work in parallel, each checking a group. Same accuracy, 4x throughput. The guards share a walkie-talkie (shared memory) to coordinate.

QPU (Quantum X-ray scanner) — Scans everyone with quantum patterns that can see invisible differences between fans and hackers. But at this small concert (120 people), the scanner alone only catches 62.5% — it needs more data to shine.

The Hybrid Team — CPU cleans the data (removes noise), GPU extracts deep features (like a forensics lab), QPU finds quantum patterns. Together: 100% catch rate. The lesson? QPU alone at small scale struggles, but the team is unbeatable.

TL;DR: At 120 people, CPU wins alone. At 1,000,000 network packets per second (real cybersecurity), the quantum kernel sees attack patterns that classical ML misses entirely.
Our demo: 120 samples, 4 features Real world: millions of packets/sec, 100+ features (Darktrace, CrowdStrike)

City Planner: Secure the Network
Quantum Approximate Optimization (QAOA)

You're a city planner with 7 buildings connected by 9 roads. You need to split the city into two security zones so the maximum number of roads cross the boundary — easier to put checkpoints on those roads.

CPU (Greedy planner) — Looks at each building one at a time and puts it in whichever zone creates more border roads. Quick decision, sometimes gets lucky, sometimes misses the best answer.

GPU (Math department) — Calculates a giant 128x128 matrix that represents ALL possible ways to split the city. Powerful analysis, but can't make the final decision alone.

QPU (Quantum city planner) — Puts all 128 possible arrangements into quantum superposition — exploring them ALL at once. Then runs optimization waves (QAOA layers) to find the best one. At 7 buildings it gets ~76% of optimal; at 50+ buildings, it crushes classical methods.

The Hybrid Team — CPU generates the map, GPU builds the math, QPU explores quantum solutions. CPU also provides a greedy baseline to compare against.

TL;DR: With 7 buildings = 128 arrangements (easy). With 50 buildings = 1,125,899,906,842,624 arrangements (impossible classically). QAOA explores them all simultaneously.
Our demo: 7 nodes, 128 arrangements Real world: 1000+ node networks (Cisco, BMW supply chain, JP Morgan portfolios)

The Detective Agency: Full Mission
Complete Hybrid CPU + GPU + QPU Pipeline

A cybersecurity company gets a red alert: suspicious network activity detected! They deploy their full hybrid team on three missions simultaneously:

CPU (The Detective) — Collects all the evidence. Organizes network logs, cleans the data, runs quick classical checks. Also serves as the boss who writes the final report.

GPU (The Forensics Lab) — Runs thousands of parallel tests on the evidence. Extracts deep features, builds mathematical models, crunches huge matrices. The muscle of the operation.

QPU (The Quantum Analyst) — Uses quantum algorithms for the hardest tasks: quantum kernel analysis to catch disguised hackers (QSVM), quantum optimization to find the best network defense (QAOA), and quantum search to crack suspicious encryption keys (Grover).

Together — 17 tasks flow across all three units. Each does what it's best at. The result: faster detection, better accuracy, comprehensive security analysis.

TL;DR: This is the full thesis in action. CPU handles 8 tasks, GPU handles 6, QPU handles 3. Each unit plays its role. This is how real hybrid quantum-classical systems will work in production.
